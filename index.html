漫谈：C数组与指针

发表于 2021-11-11 更新于 2021-11-15

C 语言中只有一维数组，而且数组的大小必须在编译器就作为一个常数确定下来。 ——《C陷阱与缺陷》

说来奇怪，虽然我自认为C语言学的还不错但是却还是有些忽略的地方。当初考试的C语言成绩几乎是全系最高，而且还做过不少项目，虽然最近一年多我大部分时间用的都是 Python，但是我还是对我的C语言水平非常的自满，直到我最近在一些细节上跌了跟头。起因是一道 leetcode 题目，304. 二维区域和检索 - 矩阵不可变; 我计划使用前缀和数组进行求解，但是由于我对于C语言中数组这一概念的理解存在一些问题，导致了程序出现了很大的Bug。

数组与指针

在学习C语言的过程中，常常会把数组与指针放在一起比较，似乎所有的C语言书籍都会提到它们是多么的相似，以至于我们常常会忘记它们并没有那样相同。而今天我们话题也将从这里展开。

在C语言中数组与指针是两种相容的数据类型，注意我这里并没有说它们相同，在我看来它们更像是实现了一些相同的“接口”。虽然这可能是一种面向对象的描述方式，但是用在描述数组与指针的关系上却是非常恰当。它们本来是两种完全不同的类型，只是实现了类似的加法与解引用运算符，至于下标操作，它可以看作是一种加法与解引用操作混合的语法糖。

我们常说数组名代表数组第一个元素的地址，这句话看似没有什么问题，而且不管是直接以整数的方式输出，还是将其赋值给指针类型的变量都可以这样理解。但是我认为这句话非常容易让人产生一些错误的理解，认为数组名就是一种指针变量；这是一种常见的谬误，如果你这样理解的话将无法解释下面这些问题：

  1. 使用 sizeof(数组名) 返回的是数组类型的长度（即，元素类型长度*数组长度）而非指针类型的长度。
  2. 在赋值运算中它无法成为左值，同时无法将一个指针变量赋值给它，同时也无法给它赋值另一个同类型的数组。
  3. 对数组名取地址，其值还是它本身（对于大多数早期C语言实现，这甚至是非法的）。

上面这三条性质非常特殊，尤其是第三条性质。性质1，3似乎在将讨论引向另一个结论：数组名所代表的是整个数组。但是这却又无法解释性质 2，如果真的可以代表整个数组的话那他就应该与其他的数据类型一样能够通过赋值的方式拷贝自己的值。这就是数组最奇怪的地方，对于数组名代表数组第一个元素的地址最好的理解应该是：数组名类似于一种被强制转换成指针类型的数组第一个元素的地址的字面量。

值得注意的是，我认为之所以着重强调字面量，是为了区分C语言中的常变量。对于C语言中的常变量指的是其在定义过程中使用了 const 关键字修饰，它们虽然无法改变，但是可以被取地址；而在实际上的汇编代码中程序并不会另外开辟空间存储数组的首地址，而是以立即数的方式使用数组名。

数组的指针

在讨论完数组与指针后，我们再来讨论一下数组和指针。上面说过：对数组名取地址，其值还是它本身。这可以参考其他组合类型如结构体，组合类型的地址为组合类型中首元素的地址；所以对于数组而言其变量名就代表首元素地址，其取地址后值不变，但是数据类型发生了变化，这也解释了为什么数组与结构体之间可以非常完美进行强制类型转换；同时在汇编语言的角度上结构体与数组的访问方式几乎一样。
